# 사내 메신저 MVP

기획서(1차 MVP 범위)에 따라 구현한 실시간 메신저 수직 슬라이스입니다.

## 포함된 기능

- 이메일/비밀번호 로그인·회원가입 (JWT) — 실제 SSO/AD 연동 전 임시 인증
- 조직도 디렉터리(부서별 사용자 목록) → 클릭 시 1:1 DM 시작
- 그룹 채널 생성(이름 + 멤버 선택)
- 실시간 메시지 송수신 (Socket.io), 채널별 이력 조회
- 타이핑 표시, 온라인/오프라인 프레즌스
- 읽음 처리 및 채널별 안 읽은 메시지 뱃지
- **파일/이미지 업로드** — 이미지는 말풍선에 바로 미리보기, 그 외 파일은 다운로드 링크
- **메시지 검색** — 내가 속한 채널 전체에서 키워드 검색, 클릭 시 해당 채널/스레드로 이동
- **스레드(답장)** — 메시지에 답장하면 별도 스레드 패널에서 대화, 원본 메시지에 답장 수 표시
- **메시지 수정/삭제** — 본인 메시지만 가능, 삭제는 소프트 삭제("삭제된 메시지입니다")
- **이모지 반응** — 👍❤️😂😮😢🙏 중 선택, 실시간으로 카운트 반영
- **다크 모드** 토글 (설정은 로컬에 저장)
- 메시지 내 URL 자동 링크화
- **이전 메시지 더 불러오기** — 채널당 최근 50개만 초기 로드, 버튼으로 과거 메시지 페이지네이션
- **브라우저 데스크톱 알림** — 탭이 비활성이거나 다른 채널을 보고 있을 때 새 메시지 알림
- **그룹 채널 멤버 관리** — 만든 뒤에도 멤버 추가 가능, 본인은 채널 나가기 가능(마지막 멤버가 나가면 채널 삭제)
- **@멘션** — 입력 중 `@`로 채널 멤버 자동완성, 멘션된 이름은 강조 표시(나를 멘션하면 별도 강조)
- **출퇴근 체크인/체크아웃** — 사이드바 "출퇴근" 메뉴에서 출근/퇴근 기록, 오늘 팀 전체 현황판(누가 출근/퇴근했는지 실시간 반영), 내 최근 14일 기록 및 근무시간 계산

## 이번 범위에서 제외한 것 (다음 단계)

- 음성/영상 통화
- 실제 SSO/LDAP/AD 연동 (지금은 자체 회원가입으로 대체)
- 모바일 앱, 데스크톱(Electron) 패키징
- AI 기능(요약/스마트 리플라이 등)
- 관리자 콘솔 + 감사 로그
- 근태 관리자용 통계/엑셀 export, 지각·조퇴 정책 설정

## 기술 스택

| 영역 | 선택 |
|---|---|
| 서버 | Node.js + Express + Socket.io |
| 저장소 | SQLite (`better-sqlite3`, WAL 모드) — 별도 DB 서버 설치 없이 바로 실행. 운영 배포 시 PostgreSQL로 교체 예정(기획서 Phase 2) |
| 파일 업로드 | multer (로컬 디스크 `server/uploads/`, 정적 서빙) |
| 인증 | JWT (jsonwebtoken) + bcryptjs |
| 클라이언트 | React + TypeScript + Vite, socket.io-client |

## 실행 방법

### 1. 서버

```bash
cd server
npm install
npm run seed   # 데모 조직도 계정 생성 (최초 1회)
npm run dev    # http://localhost:4000
```

시드 계정 (비밀번호 공통 `password123`):

- admin@example.com (김관리, 경영지원팀)
- dev1@example.com (이개발, 개발팀)
- dev2@example.com (박코딩, 개발팀)
- design1@example.com (최디자인, 디자인팀)
- sales1@example.com (정영업, 영업팀)

### 2. 클라이언트

```bash
cd client
npm install
npm run dev    # http://localhost:5173
```

브라우저 두 개(또는 시크릿 창)를 열어 서로 다른 계정으로 로그인하면 실시간 메시징을 바로 확인할 수 있습니다.

## 환경 변수

- 서버: `JWT_SECRET`(기본값은 개발용이며 운영 배포 전 반드시 교체), `PORT`(기본 4000), `CLIENT_ORIGIN`(기본 `http://localhost:5173`)
- 클라이언트: `VITE_API_URL`(기본 `http://localhost:4000`)

## 알려진 이슈 / 제약

- `client`의 `vite`/`esbuild` 개발 서버에 알려진 CORS 취약점(GHSA-67mh-4wv8-2f99)이 있습니다. 로컬 개발 서버에만 영향을 주며 프로덕션 빌드에는 영향 없습니다. 메이저 업그레이드(`vite@8`)가 필요해 별도 승인 없이 자동 반영하지 않았습니다.
- SQLite 파일 기반 저장소라 다중 서버 인스턴스 확장(수평 확장)에는 적합하지 않습니다. 실사용 규모가 커지면 PostgreSQL 전환이 필요합니다.
- 파일 업로드는 로컬 디스크에 저장됩니다. 실제 배포 시 S3 등 오브젝트 스토리지로 교체 필요, 업로드 용량 상한은 20MB로 하드코딩되어 있습니다.
- 검색은 SQL `LIKE` 기반 단순 부분일치입니다. 데이터가 많아지면 FTS5 등 전문 검색으로 전환을 고려해야 합니다.
- 스레드는 1단계 깊이만 지원합니다(답장에 다시 답장 불가) — Slack과 동일한 제약.
- 브라우저 알림은 "새 메시지 전체" 기준입니다. 멘션된 경우만 알림을 받는 세밀한 설정은 아직 없습니다.
- 그룹 멤버 삭제(추방)는 아직 없습니다 — 본인이 나가는 것만 가능합니다.
- 출퇴근 날짜는 서버가 실행되는 호스트의 로컬 타임존 기준입니다(사용자별 타임존 미지원).
